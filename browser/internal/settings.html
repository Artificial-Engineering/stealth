Internet

The Stealth Browser can connect to a website through direct internet access machines and peer-to-peer internet access machines.

If this machine does not have internet access whatsoever, select the peer-to-peer option. It will get internet access once a configured peer is found in the local network.

The Stealth Browser supports using a TOR Proxy to access the Deep Web.

If a TOR proxy is used, local peer discovery doesn't work anymore, so this machine has to have mobile or broadband internet access in order to connect to sites on the internet.

Filters

The Stealth Browser filters websites, which means that all advertisements, annoyances and known malware sites embedded in a website are blocked.

As the web keeps changing and malware developers keep changing changing their domains and websites - you need to regularly update these filter lists.

Hosts

The Stealth Browser has its own DNS resolution cache in order to prevent tracking capabilities of Internet Service Providers when being online via 2G/3G/4G.

If a domain has an AAAA entry and supports IPv6, Stealth Browser will connect to it via IPv6.

It is possible to use transparent IPv6 proxies here, so for example when github.com only supports IPv4 and ikvjwd.com supports IPv6, you can manually edit the IPv6 of github.com and use the one of the IPv6 proxy.

Peers

The Stealth Browser supports Peer-to-Peer transfer of already archived and cached resources among its trusted Peers.

The Mode for each Peer influences what types of resources can be exchanged. If you're unsure what Modes do, take a look at the stealth:welcome page.

Domain Capacity Mode Actions

Plugins

The Stealth Browser offers a Plugin API that allows to more efficiently query websites and their data using their API instead of downloading and filtering the HTML file and its linked resources.

If a plugin is selected, it will be used for every request that leads to its matching domain.

Sites

The Stealth Browser has a Stealth Mode which will influence on what is loaded by default. If a Domain has a Site-specific Mode set, it is incrementally added to the default Stealth Mode.

That means if you do not trust a Domain anymore, you need to remove its Site-specific Mode setting. On the other hand if you trust a Domain, you can override its settings by using this list here.
